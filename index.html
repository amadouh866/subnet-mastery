4 Techniques

Master these in order. Technique 1 alone solves 80% of exam questions.

01
Block Size Shortcut

Fastest method for CIDR ≥ /24. Works in your head in seconds.

  1. Find the interesting octet (not 255 or 0 in the mask)
  2. Block size = 256 − mask octet
  3. List multiples of block: 0, B, 2B, 3B…
  4. IP's octet falls between two multiples → that's your subnet
  5. Broadcast = next multiple − 1
Example
IP: 192.168.10.77 /27 → mask = 224
Block = 256−224 = 32
Multiples: 0, 32, 64, 96… → 77 in 64–95
Net: 192.168.10.64 | BC: 192.168.10.95
02
Count Host Bits

Derive the mask from a prefix number instantly.

  1. Host bits h = 32 − prefix
  2. Total addresses = 2ʰ, usable = 2ʰ − 2
  3. Zero out h bits from the right of the mask
Example
/26 → h = 32−26 = 6 host bits
2⁶ = 64 addresses → 62 hosts
Mask last octet: 11000000 = 192
03
Fit N Hosts

"Need N hosts per subnet" — work backwards from the host count.

  1. Find smallest 2ʰ ≥ N + 2
  2. That gives you h (host bits needed)
  3. Prefix = 32 − h
Example
Need 50 hosts → 50+2=52
Smallest 2ʰ ≥ 52 → 2⁶=64 → h=6
Prefix = 32−6 = /26 → 62 hosts ✓
04
Fit N Subnets

"Need N subnets" — borrow bits from the host side.

  1. Find smallest 2ˢ ≥ N
  2. That gives you s (bits to borrow)
  3. Add s to your starting prefix
Example
Need 6 subnets from /24
2ˢ ≥ 6 → 2³=8 → borrow 3 bits
New prefix: /24+3 = /27 → 8 subnets ✓
